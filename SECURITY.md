# Security Policy

## Supported versions

This repository currently supports security fixes for the **latest** version available on the `main` branch.

## Reporting a vulnerability

If you believe you’ve found a security vulnerability, please report it **privately**.

- Do **not** open a public GitHub issue for security reports.
- Include as much detail as possible: reproduction steps, impacted areas, and any proof-of-concept.

### Where to report

- If this repository is part of an organization, use the organization’s preferred private reporting channel.
- Otherwise, open a **private** security advisory in GitHub (if enabled) for this repository.

## What to expect

- **Acknowledgement**: within 3 business days
- **Initial assessment**: within 7 business days
- **Fix / mitigation**: timeline depends on severity and scope

## Scope

In-scope examples:

- Remote code execution, privilege escalation, or sandbox escape
- Sensitive data exposure (credentials, tokens, secrets)
- Authentication/authorization bypasses
- Dependency supply-chain vulnerabilities that directly impact the app

Out of scope examples:

- Issues that require physical access to an unlocked device
- Social engineering attacks
- DoS that requires unrealistic conditions
